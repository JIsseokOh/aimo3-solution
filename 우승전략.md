## Phase 2: 약점 분석

### 목표
- reference.csv 10문제로 현재 모델의 정확도 측정
- 오류 패턴 분석 (어떤 유형의 문제를 틀리는지)

### 작업
1. reference.csv 문제 분류
   - 대수학 (Algebra)
   - 조합론 (Combinatorics)
   - 기하학 (Geometry)
   - 정수론 (Number Theory)

2. 각 유형별 정확도 측정

3. 오류 분석
   - 계산 오류
   - 논리 오류

   - 코드 실행 오류

### 예상 시간: 2-3일

## Phase 3: 데이터 수집 및 구축

### 목표
- 고품질 수학 올림피아드 학습 데이터 확보

### 데이터 소스
1. **공개 데이터셋**
   - MATH dataset (Hendrycks et al.)
   - GSM8K
   - AIME problems
   - IMO Shortlist (1959-2023)
   - 국가올림피아드 문제 (KMO, USAMO, etc.)

2. **합성 데이터 생성**
   - GPT-4/Claude로 문제 변형
   - 풀이 과정 생성
   - 검증 후 필터링

3. **데이터 형식**
   ```json
   {
     "problem": "문제 텍스트",
     "solution": "단계별 풀이",
     "code": "검증 코드",
     "answer": 정답,
     "category": "algebra/combinatorics/geometry/number_theory",
     "difficulty": "national/imo"
   }
   ```

### 목표 데이터 규모
- 최소 10,000개 고품질 문제-풀이 쌍
- IMO 수준 문제 1,000개 이상

### 예상 시간: 1-2주

## Phase 4: 파인튜닝

### 목표
- Qwen2.5-Math-72B를 IMO 수준 문제에 특화

### 방법
1. **LoRA/QLoRA 파인튜닝**
   - 메모리 효율적
   - 빠른 학습
   - 원본 모델 보존

2. **학습 설정**
   ```python
   training_args = {
       "learning_rate": 2e-5,
       "num_train_epochs": 3,
       "per_device_train_batch_size": 4,
       "gradient_accumulation_steps": 8,
       "lora_r": 64,
       "lora_alpha": 128,
   }
   ```

3. **학습 환경**
   - Kaggle H100 128대 (Fields Model Initiative)
   - 또는 자체 GPU 클러스터

### 예상 시간: 3-5일

---

## Phase 5: 추론 고도화

### 목표
- 정확도 극대화를 위한 추론 전략 최적화

### 기법
1. **Multi-Model Ensemble**
   - Qwen2.5-Math-72B
   - DeepSeek-Math-7B-RL
   - NuminaMath-7B-TIR
   - 다수결 또는 가중 투표

2. **Monte Carlo Tree Search (MCTS)**
   - 여러 추론 경로 탐색
   - 가장 유망한 경로 선택

3. **Self-Consistency 강화**
   - 샘플 수 최적화
   - Temperature 스케줄링

4. **Multi-turn Verification**
   - 1차 답안 도출
   - 검증 프롬프트로 재확인
   - 오류 발견 시 재풀이

5. **Tool-Integrated Reasoning (TIR)**
   - Python 코드로 계산 검증
   - SymPy로 기호 연산
   - 기하 문제용 라이브러리

### 예상 시간: 1주
